Master Project: Automated QoS Data Processing System

January 2025 - June 2025 | Final Internship at Business Intelligence for Telecommunication (BI4T)

Developed a comprehensive system to automate the processing of Quality of Service (QoS) data for 2G, 3G, 4G, and 5G networks, significantly enhancing efficiency and reliability for telecom regulators and operators. The system transforms raw data into actionable insights through advanced dashboards, geospatial mapping, and automated reporting.

Key Achievements

  • Designed and developed a Java/Spring Batch pipeline to process raw mobile network performance data from multi-format files (.csv, .xml, .data), reducing manual effort by 90%.
  • Architected a PostgreSQL/PostGIS database for storing and analyzing geospatial KPIs (e.g., latency, coverage), enabling spatial queries for infrastructure optimization.
  • Automated ETL tasks to calculate hourly, daily, and weekly KPIs, including critical metrics like Call Setup Success Rate (CSSR), Call Drop Rate (CDR), and Handover Success Rate (HOSuccess), providing real-time QoS dashboards to operators.
  • Implemented validation checks and error handling, achieving 99.5% system reliability, with detailed logging for transparency and troubleshooting.

Technical Details

KPI Calculation: The system calculates KPIs using standardized formulas, such as CSSR = (Successful Call Setups / Total Call Attempts) * 100 and CDR = (Dropped Calls / Total Call Attempts) * 100. It stores numerator and denominator values (e.g., n_cssr, d_cssr) for flexibility, handling non-calculable cases with attributes like isCalculable to ensure data integrity across diverse network data from vendors like Huawei and Ericsson.

Interactive Dashboard: The dashboard offers real-time KPI displays with customizable views (by region, period, or cell), drill-down capabilities, and threshold alerts. It features circular gauges for CSSR (60%), CDR (0%), TauxDispo (100%), and HOSuccess (40%), alongside a trends graph to identify anomalies like a CSSR drop at 02:00, enhancing decision-making for network performance monitoring.

Coverage Map: The geospatial coverage map visualizes KPIs like signal strength and data speeds using color-coded heatmaps, with layering options for different operators and network technologies (2G-5G). It supports zoom/pan functionality and point-and-click details, identifying coverage gaps for strategic planning and network expansion.

Report Generation: The system generates customizable reports (daily, weekly, monthly) in PDF or CSV formats, incorporating visualizations and aggregated KPIs. Users can select specific regions and time periods, with the process leveraging pre-calculated data from the PostgreSQL database for efficiency, optimized by tables like table_valeurs_mois.

Technologies: Java Spring Batch PostgreSQL PostGIS Bash Cron
