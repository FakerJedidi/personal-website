F1 Race Winner Prediction Project

Individual Project

Developed a Python-based system to predict Formula 1 race winners for the 2025 season using historical race data and machine learning. The project leverages the FastF1 library to access race data and the OpenWeatherMap API for weather conditions, predicting average lap times for drivers and determining the likely race winner. The system includes interactive features for users to select races and visualize predictions through scatter plots and feature importance charts.

View the project on Kaggle

Key Achievements

  • Designed a data pipeline to process 2024 F1 race data from FastF1, handling missing values and integrating weather data for accurate predictions.
  • Implemented a Gradient Boosting Regressor to predict average lap times, achieving a mean absolute error (MAE) of approximately 0.07 seconds, with dynamic race selection for 2025 races.
  • Created interactive visualizations using Matplotlib, including scatter plots of clean air race pace vs. predicted lap times and feature importance plots to highlight key predictors like qualifying time and team performance.
  • Developed an intuitive interface allowing users to input race names (e.g., Bahrain, Monaco) and view predicted podiums with full driver names and estimated total race times.

Technical Details

Data Processing: The pipeline retrieves 2024 race data (e.g., lap times, sector times) from the FastF1 library and processes it with Pandas. Features include qualifying times, clean air race pace, team performance scores, weather conditions (rain probability, temperature), and track-specific metrics like average position change. Missing values are handled using median imputation with Scikit-learn’s SimpleImputer.

Predictive Model: A Gradient Boosting Regressor (n_estimators=200, learning_rate=0.1, max_depth=4) predicts average lap times based on nine features, including qualifying time, team performance, and weather factors. The model is trained on 2024 race data with a 70-30 train-test split (random_state=38), achieving high accuracy (MAE ~0.07 seconds). Total race times are estimated by multiplying predicted lap times by the number of laps and adding pit stop times (assumed 44 seconds for two stops).

Visualization: Interactive plots include scatter plots of clean air race pace vs. predicted lap times (annotated with driver names) and horizontal bar charts showing feature importance. These visualizations help users understand the model’s decision-making process and the impact of features like qualifying time and weather.

User Interface: The system prompts users to select a race (e.g., Bahrain, Monaco) and uses predefined race data (latitude, longitude, lap counts) to fetch weather data and make predictions. Outputs include a ranked list of drivers, predicted podiums, and estimated total race times in minutes.
